## P0 — 严重问题 (CRITICAL): 1 项

### P0-1: ProjectCard 嵌套交互元素 — 键盘陷阱风险

**文件**: `src/components/ProjectCard.tsx:66-97`
**参考**: UI/UX Pro Max UX — "Keyboard Navigation" + "Semantic HTML"

**问题**: 整个卡片 `div` 设置了 `role="button"` + `tabIndex={0}`，同时内部「打开项目」叠加层也包含一个 `span`。这造成键盘 Tab 顺序不清晰，Enter 触发卡片 URL 打开，但叠加层按钮文字暗示它是主交互。

**修复**:
```tsx
// 叠加层 span 改为 div + aria-hidden="true"
<div aria-hidden="true" className="opacity-0 group-hover:opacity-100">
  <span className="flex items-center gap-2 ...">
    打开项目
    <svg aria-hidden="true" ... />
  </span>
</div>
```

## P1 — 高优问题 (HIGH): 3 项

### P1-1: 精选标签 `.visible_&` 选择器过于脆弱

**文件**: `src/components/ProjectCard.tsx:99`

**问题**: Tailwind 4 arbitrary variant `[.visible_&]:opacity-100` 依赖父选择器 `.visible`，将来可能误匹配。

**建议**: 改用 CSS `@keyframes` + `animation-fill-mode: forwards`。

---

### P1-2: 占位符卡片缺少视觉禁用状态

**文件**: `src/components/ProjectCard.tsx:66`

**问题**: 占位符卡片只设置 `cursor-default`，没有降低透明度/灰度等禁用提示。

**建议**:
```tsx
className={`... ${isPlaceholder ? 'opacity-60 grayscale-[0.5] cursor-not-allowed' : 'cursor-pointer hover:...'}`}
```

---

### P1-3: 缺少 `theme-color` Meta 标签

**文件**: `index.html`

**问题**: 没有 `theme-color` meta，Android Chrome 地址栏不会使用品牌颜色。

**修复**:
```html
<meta name="theme-color" content="#0a0a0c" />
```

## P2 — 建议优化 (MEDIUM): 3 项

### P2-1: FilterBar 筛选组建议用 `<nav>`

**文件**: `src/components/FilterBar.tsx:9`
**当前**: `<div role="group" aria-label="标签筛选">`
**观察**: `role="group"` 对按钮组语义足够，但 `<nav>` 更符合 HTML5 规范。保持现状亦可。

---

### P2-2: Bundle 208KB 无代码分割

**观察**: 208KB (gzip: 66KB)，当前可接受。卡片数量 > 20 时考虑 `React.lazy`。

---

### P2-3: Hero 动画 nth-child 选择器耦合

**文件**: `src/index.css:167-169`

**观察**: Hero Section 内 3 个 `.hero-animate` 依赖 DOM 顺序。建议用内联 `style` 或 `nth-of-type`。
