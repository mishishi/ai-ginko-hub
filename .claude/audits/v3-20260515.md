## P1 — 设计系统对齐: 2 项观察

### P1-1: 搜索框 — aria-label vs 显式 label

**发现**: 搜索输入框 `<input aria-label="搜索项目">` 但无可见 label 元素。

**参考**: UI/UX Pro Max React 栈明确指出 "Placeholder as only label" 是 `Don't` 模式。

**现状**: 功能正常（屏幕阅读器可识别），但不符合最佳实践。

**建议**: 添加 `<label htmlFor="search-input" className="sr-only">搜索项目</label>` 并将 id 加到 input。

**状态**: 建议优化，非阻塞。

### P1-2: 项目卡片堆叠上下文

**发现**: 卡片使用 `hover:z-10`，父容器 `<main>` 无 z-index 声明。

**参考**: UI/UX Pro Max UX 指南 — Stacking Context: "Parent with z-index isolates children"。

**分析**: 卡片在 `hover:z-10` 时会创建新的堆叠上下文，但 `<main>` 无 z-index，其子元素不会与 header (z-50) 冲突。当前实现安全。

**状态**: ✅ 确认合理。
