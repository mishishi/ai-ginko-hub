## 问题清单

> [!TIP]
> 按严重程度分级：
> - 🔴 P0：必须修复，影响核心体验或无障碍合规
> - 🟡 P1：强烈建议修复，影响整体品质感
> - 🟢 P2：优化建议，可提升但不影响使用

---

### 🔴 P0 — 必须修复

#### P0-1: 移动端 Header 高度过大

**位置**: `src/components/Header.tsx:9`
**问题**: `h-[72px]` 在移动端（375px）占视觉空间过多，典型移动 nav 高度为 56px。
```tsx
// 当前
className="... h-[72px] ..."
// 建议
h-14 sm:h-[72px]
```
**影响**: 移动端首屏可见项目数量减少，体验下降。

---

#### P0-2: 搜索框在移动端宽度不足

**位置**: `src/components/Header.tsx:26`
**问题**: `max-w-80` 在小屏设备上导致搜索框过窄，文字易截断。
```tsx
// 当前
className="... max-w-80 ..."
// 建议
max-w-48 sm:max-w-80
```
**影响**: 移动端搜索体验差，输入内容易被截断。

---

#### P0-3: 移动端 FilterBar 按钮可读性差

**位置**: `src/components/FilterBar.tsx:12-17`
**问题**: 移动端标签按钮过多换行（`flex flex-wrap`），且 `min-h-[44px]` 在竖屏时字号偏小（`text-xs`）。
```tsx
// 当前
className={`... text-xs ...`}
// 建议
text-[11px] sm:text-xs
```
**影响**: 移动端标签区占据过多垂直空间（6 行标签），首屏效率低。

---

### 🟡 P1 — 强烈建议

#### P1-1: Hero 区域移动端行间距过紧

**位置**: `src/App.tsx:63-88`
**问题**: `pt-24` 和 Hero 文字行高（tailwind 默认 1.5）导致移动端首屏拥挤。统计数字与标签之间缺少视觉分隔。
```tsx
// 当前
className="... pt-24 pb-16 ..."
// 建议
pt-16 sm:pt-24 pb-10 sm:pb-16
```
**影响**: 移动端 Hero 区域视觉压力较大。

---

#### P1-2: ProjectCard 移动端悬停行为无意义

**位置**: `src/components/ProjectCard.tsx:55`
**问题**: 项目卡片使用 `hover:-translate-y-1.5` 等悬停效果。移动端无 hover 概念，这些样式在触摸时无效但仍消耗渲染。

---

#### P1-3: Banner 资源存在但未被引用

**位置**: `public/banner.svg`
**问题**: 生成了 SVG banner 但 README 未使用（已撤销），且项目未使用 og-image 作为社交分享图。

**建议**:
1. 将 `public/og-image.svg` 或 `public/pixel-banner.svg` 作为 `<meta property="og:image">` 引用于 `index.html`
2. 在 `index.html` 添加 Twitter Card 标签
```html
<meta property="og:title" content="Ginko Hub — AI Project Showcase" />
<meta property="og:description" content="AI 辅助开发的 Web 项目合集" />
<meta property="og:image" content="/og-image.svg" />
<meta name="twitter:card" content="summary_large_image" />
<meta name="twitter:image" content="/pixel-banner.svg" />
```

---

#### P1-4: Footer 在移动端过于简单

**位置**: `src/App.tsx:114-127`
**问题**: Footer 只有两行文字，在大屏上显得空洞。缺少社交链接、品牌标识或版权年份。
```tsx
// 当前
<p className="text-xs text-text-muted tracking-wide">Ginko Hub — Built with AI</p>
<a href="https://github.com">GitHub</a>
// 建议添加：Copyright 年份、更多社交链接、内部导航
```

---

#### P1-5: 空状态缺少图标尺寸规范

**位置**: `src/components/ProjectGrid.tsx:12`
**问题**: 空状态 SVG `width="48"` 硬编码，`mb-4` 与邻近元素间距比例未遵循 8px 网格。

---

#### P1-6: 版本号占位符

**位置**: `public/banner.svg`
**问题**: 版本号 `v1.0.0` 是硬编码字符串，每次发版需要手动更新。考虑通过构建工具自动注入。

---

### 🟢 P2 — 优化建议

#### P2-1: 动画时长可微调

**位置**: `src/components/ProjectCard.tsx:55`, `src/index.css:141-148`
**问题**: 滚动渐入动画 `transition: opacity 0.5s` + `transform 0.5s`。Pro Max 建议 micro-interaction 150-300ms。

#### P2-2: 缺少 loading skeleton

**位置**: 全站
**问题**: 当前无网络请求，但未来数据来自 API 时没有 skeleton 状态设计。

#### P2-3: 标签颜色 Map 未扩展

**位置**: `src/components/ProjectCard.tsx:20-45`
**问题**: `tagColors` 字典未覆盖所有项目标签（如 `SaaS`、`Media`、`Low-Code` 等），fallback 到 `#6b6865` 与默认文字颜色 `#8a8784` 接近，对比度不足。

#### P2-4: 项目描述文本截断

**位置**: `src/components/ProjectCard.tsx:96-97`
**问题**: `line-clamp-2` 在描述超过 2 行时截断，可能丢失关键信息。可考虑在悬停时显示更多，或改为 3 行截断。

#### P2-5: 缺少全局滚动进度条

**位置**: 全站
**问题**: 内容丰富的展示站，滚动进度条可增强导航感知。
