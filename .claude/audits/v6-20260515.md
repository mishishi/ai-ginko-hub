## 新发现: 3 项高优问题

### P1-1: URL 同步使用 `replaceState` — 回退按钮失效

**文件**: `src/App.tsx:23`
**参考**: UI/UX Pro Max UX — "Back Button: Users expect back to work predictably" + "Deep Linking: URLs should reflect current state"

**问题**: 筛选/搜索状态使用 `window.history.replaceState()` 同步到 URL。这意味着：
- 用户点击标签筛选 → URL 变为 `?tag=React`
- 用户点击「返回」按钮 → 直接返回浏览器历史上一页（上一个页面），而非恢复到「全部」状态
- 筛选状态无法通过 URL 分享和恢复

**当前**:
```tsx
window.history.replaceState(null, '', url);
```

**修复**:
```tsx
window.history.pushState(null, '', url);
```

同时需要添加 `popstate` 监听以支持浏览器后退按钮恢复筛选状态：
```tsx
useEffect(() => {
  const onPopState = () => {
    const params = new URLSearchParams(window.location.search);
    setSearchQuery(params.get('q') || '');
    setActiveTag(params.get('tag') || null);
  };
  window.addEventListener('popstate', onPopState);
  return () => window.removeEventListener('popstate', onPopState);
}, []);
```

### P1-2: Footer 社交链接缺少 `<nav>` 语义标签

**文件**: `src/App.tsx:127-140`
**参考**: UI/UX Pro Max UX — "Screen Reader: Content should make sense when read aloud" + "Semantic HTML: Use nav for navigation sections"

**问题**: Footer 中的 GitHub 链接组是导航链接，但使用的是裸 `<div>` + `<a>`，缺少 `<nav>` 语义容器。屏幕阅读器无法将其识别为导航区域。

**当前**:
```tsx
<div className="flex items-center gap-4">
  <a href="https://github.com" ...>GitHub</a>
</div>
```

**修复**:
```tsx
<nav aria-label="社交链接">
  <div className="flex items-center gap-4">
    <a href="https://github.com" ...>GitHub</a>
  </div>
</nav>
```

### P1-3: Google Fonts 缺少 `font-display=swap`

**文件**: `index.html:22`
**参考**: UI/UX Pro Max UX — "Font Loading: Web fonts can block rendering" + "Use font-display swap or optional"

**问题**: Google Fonts URL 未指定 `font-display=swap`，浏览器可能先显示不可见文字（FOIT），或字体加载后产生闪烁（FOUT）。虽然当前有 fallback 字体，但最佳实践是明确声明。

**当前**:
```html
<link href="...family=Outfit:wght@...&family=Playfair+Display:...&display=swap" rel="stylesheet" />
```

**注意**: URL 中已有 `display=swap` 参数（`&display=swap`）——需确认是否在 URL 中。检查发现 URL 没有 `display=swap` 参数。

**修复**: 在 Google Fonts URL 末尾添加 `&display=swap`：
```html
<link href="...family=Outfit:wght@...&family=Playfair+Display:...&display=swap" rel="stylesheet" />
```

## 建议优化 (P2): 2 项

### P2-1: Bundle 大小 207KB 无代码分割

**参考**: UI/UX Pro Max React — "Lazy load components: Use React.lazy for code splitting"

**观察**: 当前构建产物 JS: 207KB (gzip: 66KB)，所有代码打包为单个 bundle。`ProjectCard` 使用 `useScrollReveal` hook，组件整体直接 import。对于展示类站点当前体积可接受，但随着项目增多建议考虑：

```tsx
// 未来可优化方向
const ProjectCard = lazy(() => import('./ProjectCard'));
```

**状态**: 当前可接受，观察处理。

### P2-2: FilterBar 筛选组缺少 `<nav>` 语义

**文件**: `src/components/FilterBar.tsx:9`

**当前**:
```tsx
<div role="group" aria-label="标签筛选">
```

**观察**: `role="group"` 是 ARIA 语义，但不是原生 HTML `<nav>`。对于屏幕阅读器，`role="group"` 足够表示「这是一个相关按钮组」，但 `<nav>` 语义更强。考虑到这是页面内导航（非站点级导航），`role="group"` 可接受，保持现状亦可。

## 审计结论

**本轮新增 P1**: 3 项 — URL 回退 (`replaceState`→`pushState`)、Footer `<nav>`、Google Fonts `display=swap`

**v5 问题复核**: P1-1 (inputmode) ✅, P1-2 (aria-live) ✅ 已确认修复

**Pre-Delivery**: 25/25 ✅ — 全部通过

**风险评估**:
- P1-1 (回退按钮) — **高优**：用户通过筛选探索时，误点返回会直接离开页面而非回到上一步筛选，损害浏览流畅性
- P1-2 (Footer nav) — **中优**：无障碍体验瑕疵，不阻塞发布
- P1-3 (font-display) — **低优**：FOUT 偶发，不阻塞发布

---

**建议优先修复 P1-1** (URL 回退)，其余可按迭代处理。
