## P0 — 严重问题 (CRITICAL): 3 项

### P0-1: Header Logo SVG 缺少 aria-hidden

**文件**: `src/components/Header.tsx:13`
**问题**: Logo SVG 位于带 `aria-label="Ginko Hub 首页"` 的 `<a>` 标签内，SVG 本身是装饰性内容，需 `aria-hidden="true"` 避免屏幕阅读器重复朗读。

**当前**:
```tsx
<a href="/" aria-label="Ginko Hub 首页">
  <span>
    <svg width="28" height="28" ...>  {/* 缺少 aria-hidden */}
```

**修复**:
```tsx
<svg width="28" height="28" viewBox="0 0 32 32" fill="none" aria-hidden="true">
```

## P1 — 高优问题: 1 项

### P1-1: ProjectCard `role="button"` 与 `role="article"` 混用

**文件**: `src/components/ProjectCard.tsx:73`
**问题**: 条件渲染 `role={isPlaceholder ? 'article' : 'button'}` — 占位符时用 `article` 是为了表示「即将上线」非交互内容，但实际组件仍然可 Tab 聚焦（`tabIndex={isPlaceholder ? -1 : 0}`），且有 `onClick` 处理。

**建议**: 统一用 `role="article"`，并确保 `tabIndex` 和 `onClick` 在占位符时完全禁用。

**当前**:
```tsx
role={isPlaceholder ? 'article' : 'button'}
tabIndex={isPlaceholder ? -1 : 0}
// 但 isPlaceholder 时 onClick 仍然存在，只是不跳转
```

**分析**: 功能正确（isPlaceholder 时 onClick 为空操作），但语义可更精确。建议保持现状或统一为 `role="article"` 并确保占位符时完全无交互。
